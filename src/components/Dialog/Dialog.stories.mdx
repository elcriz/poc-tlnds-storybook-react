## Overview

A dialog is used to convey (extra) information to the user. It is up to the user to decide what to do with that information. Dialogs typically contain no triggerable actions.

### When to use this component

Use dialogs to convey information that requires special attention like success, warning or error messages. Typically, it is then up to the user what to do with that information.

## Component API

<Props />

### `id`

You can optionally set the dialog's ID by providing the optional `id` prop.

```jsx
<Dialog id="myDialog">
    Dialog with custom ID
</Dialog>
```

### `className`

You can optionally set a custom (CSS) classname for the dialog by providing the optional `className` prop.

```jsx
<Dialog className="my-dialog">
    Dialog with custom classname
</Dialog>
```

### `hasIcon`

Provide the `hasIcon` prop as  `true` to render an icon, matching the dialog's variant.

```jsx
<Dialog hasIcon>
    Dialog with an SVG icon
</Dialog>
```

## Dialog variants
