## Overview

A collection is used to display a group of related items. Because all the items inside the collection will scale responsively in the exact same way, the user will understand that these items belong together.

### When to use this component

Use a collection if you want to display a group of related items (e.g. news or blog articles) that need to scale the same way (e.g. each item at one third the size of the entire collection).

### When to not use this component

Do not use a collection if you want to create a layout with related and/or unrelated items. Also, do not use collections for grid styling purposes for any type of (related/unrelated) content. You may use the `Layout` component for that. Layouts on the other hand can be used in conjunction with collections. See the corresponding documentation for that.

## Component API

<Props />

### `id`

You can optionally set the collections's ID by providing the optional `id` prop.

```jsx
<Collection id="myRating">
    <div>Some child item</div>
</Collection>
```

### `className`

You can optionally set a custom (CSS) classname for the collection by providing the optional `className` prop.

```jsx
<Collection className="my-rating">
    <div>Some child item</div>
</Collection>
```

### `variant`

You may set the collection variant by providing its `variant` prop. The variants determine how many child items inside the collection will be displayed in a row when the screen viewport is at its widest. Please note that when the screen size changes, the collection will react responsively (e.g. with variant `oneHalf`, two items will be displayed in one row, but on smaller screens, each item will be on another row).

```jsx
<Collection variant="oneFourth">
    <div>1st child item</div>
    <div>2nd child item</div>
    <div>3rd child item</div>
    <div>4th child item</div>
    <div>5th child item</div> {/* New line */}
    <div>6th child item</div>
    <div>7th child item</div>
    <div>8th child item</div>
</Collection>
```

## Collection variants

### One fourth

On larger viewports, four child items will be displayed per row. On smaller screens (e.g. a tablet device), two items will be displayed in one row. On even smaller screens (e.g. a smartphone), one item will be displayed per row.
